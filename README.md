# Практична робота №10-11: Керування процесами та синхронізація через wait()

**Варіант №5.** Зробіть програму, в якій дочірній процес завершується з певним кодом (exit(7), наприклад), а батьківський читає цей код через wait() та виводить його.

## Опис проєкту  
Цей проєкт демонструє базові механізми керування процесами в ОС Linux. Програма на мові C використовує системні виклики для розгалуження процесів та отримання статусу завершення. Це дозволяє зрозуміти, як ієрархія процесів взаємодіє між собою та як передаються сигнали завершення від нащадків до предків.

## Якість коду  та форматування  
Для забезпечення відповідності стандартам розробки використано:
* **Стандарт іменування**: Використано змієподібний регістр (**snake_case**) для папки проєкту `lab_work_10_11_process_management`.
* **Обробка помилок**: Перевірка результату повернення `fork()` для виявлення помилок створення процесу.
* **Компіляція**: Використання **Makefile** з прапорцями `-Wall -Wextra` для суворої перевірки синтаксису.

## Логіка роботи
1. **Розгалуження**: Виклик `fork()` створює повну копію поточного процесу.
2. **Ідентифікація**: Програма аналізує повернуте значення `pid`, щоб розділити логіку виконання для батька (`pid > 0`) та дитини (`pid == 0`).
3. **Завершення нащадка**: Дочірній процес виводить свій PID і негайно завершує роботу, повертаючи цілочисельний код (наприклад, `7`) через `exit()`.
4. **Синхронізація батька**: Батьківський процес викликає `wait(&status)`, що призупиняє його виконання до моменту завершення дочірнього процесу.
5. **Декодування статусу**: Використання макросу `WEXITSTATUS(status)` для отримання чистого коду повернення з системного слова стану.

## Технічні характеристики
* **Системні виклики**: `fork()`, `wait()`, `exit()`, `getpid()`.
* **Бібліотеки**: `<unistd.h>`, `<sys/wait.h>`, `<stdlib.h>`.
* **Макроси стану**: `WIFEXITED` (перевірка нормального завершення) та `WEXITSTATUS` (отримання коду).
* **Середовище**: Linux Mint (на базі Ubuntu 22.04 LTS).

## Очікуваний результат
* Батьківський процес виведе повідомлення про те, що він чекає на дитину з конкретним PID.
* Дочірній процес виведе свій PID та інформацію про вихід.
* Батьківський процес виведе фінальне підтвердження: `Код повернення: 7`.

## Висновки  
У результаті виконання роботи було вивчено механізм передачі статусів між процесами.

* **Життєвий цикл**: Процес-батько зобов'язаний викликати `wait()`, щоб отримати статус завершення дитини. Якщо цього не зробити, завершений дочірній процес залишиться в системі у стані "зомбі" (zombie process), займаючи місце в таблиці процесів.
* **Передача даних**: Код повернення `exit()` є найпростішим способом передачі результату роботи програми (успіх/помилка) від нащадка до батька.
* **Синхронізація**: Системний виклик `wait()` забезпечує базову синхронізацію, гарантуючи, що батько не продовжить критичні операції раніше, ніж дитина завершить свою частину роботи.
